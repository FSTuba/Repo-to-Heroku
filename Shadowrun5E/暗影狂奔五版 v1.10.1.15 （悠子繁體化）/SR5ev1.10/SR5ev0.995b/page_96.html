矩陣動作 MATRIX ACTIONS

 

正如其名稱一般無可置疑的是，矩陣動作只在矩陣網絡中生效。但這並不是說在矩陣網絡中你只能進行這幾個有限的行動。矩陣動作的特殊性在於，它們應用特殊的規則。比如雜訊和監視指數（OS；Overwatch Score）。

 

這一節列舉了所有的矩陣動作。這包括了一大堆用來處理你在數字世界裡可以做的事的規則。不過當然，它們並不是你在矩陣中能進行的唯一行動。如果你想進行一些這些規則未涵蓋的行動，比如把自己的圖標改變得更時髦，或者用數據流創造出令人分心的VR影像等等，那麼你的GM會決定你需要嘗試進行何種檢定。

 

當一台設備需要進行防禦檢定，而這個檢定需要一項精神屬性的時候，使用其所有者的對應屬性等級代替。即使此時她本人並沒有參與防禦，甚至完全沒有在操作這台設備，她在這之前對設備的操作和設置已經影響了防禦檢定。如果一台設備完全處於無主狀態，它的設備等級（DR, Device Rating）代替它在防禦檢定中需要且不具有的精神屬性。舉個例子的話，一台被主人設置好，然後忘到九霄雲外的設備——比如門鎖——在對抗〈控制設備 Control Device〉行動的防禦檢定骰池中使用它的DR替代直覺屬性。

 

非法行為 ILLEGAL ACTIONS

有些矩陣動作是非法的，這讓它們比合法行為更具風險。矩陣在被建立時，安全性是考慮因素中的重中之重，但設計者當然知道不可能讓矩陣完全免遭黑客毒手。非法行為的列表非常簡單：所有攻擊性和隱匿性的動作都是非法的。非法行為的風險取決於你執行了哪一類動作。（編註：動作對應檢定的檢定上限來判斷它屬於哪類動作。比如說〈暴力破解〉是攻擊性動作，〈後台入侵〉是隱匿性動作）

 

攻擊性動作

如果你在攻擊性動作中失敗，你的目標的安全軟件會拒絕你的代碼，破壞它並把它發送回去。如果只是普通數據，你的系統會檢查錯誤，但在這種情況下，這些被設計用來繞過防火牆的代碼極其兇猛。你的目標在防禦檢定中得到的每一個淨成功，都會讓你承受1格不可抵抗的矩陣傷害。不過，因為你沒根本能影響到你的對手，所以你不會被注意到。

如果你成功地執行了一個攻擊類動作，你的目標會意識到它正在被另一個圖標攻擊。但它並不能自動地辨識到你。更大的可能性是，它會用自己的下一個動作主動搜索你的存在。然而，它總是會將此次攻擊警告給它的所有者，並且（如果是主機的話）運行IC，取決於其所有者的偏好和GM的裁斷。

 

隱匿性動作

如果你在隱匿性動作中失敗，目標的防火牆軟件會偵測到入侵並在你身上放置一個MARK。設備會立刻通知它的擁有者，主機會放出IC。如果目標在你身上已經有了3個MARK，它不會再放置更多的MARK，但通知和放出IC仍然照常。

如果你成功地執行了一個隱匿類動作，你的存在不會暴露。

 

非駭客能做的矩陣動作 MATRIX ACTIONS FOR THE NON-HACKER

你不是隊伍裡的駭客，但你的角色表有通訊鏈。你可能要問：它有什麼用？用處自然很多，但你在這章中需要記住的不多。這裡有一個你能用通訊鏈做的矩陣動作簡表。

 

你的網域 YOUR GRID

你得知道你在用什麼網域，但如果你沒有特地升級，你的網域基於你的生活方式：

低端或更低：公共網域；

中端：本地網域；

高端：全球網域——在十大巨頭裡面選一個作為你的網域提供者；

奢華：你隨便選吧，老兄。

 

你通訊鏈的矩陣屬性 YOUR COMMLINK』S MATRIX ATTRIBUTES

你的GM偶爾會問你兩項矩陣屬性：數據處理（如果你隊裡的駭客沒完成她的活）和防火牆。除非你的通訊鏈是定制的，不然這兩項屬性都等於通訊鏈的設備等級。記得在你角色單的街頭裝備部分記一下。

 

五項矩陣動作 FIVE MATRIX ACTIONS

作為一個普通的矩陣用戶，你只有五項可能需要看看的矩陣動作。

改變圖標（Change Icon）：如果你喜歡在VR模式下折騰（tooling around），你需要去看看這個動作，得到你想要的外貌。

文件編輯（Edit File）：如果你要碼字或是剪輯視頻，或者是寫報告，你可能想去瞭解一下〈編輯文件〉動作。你還可以用這個動作保護你通訊鏈上的文件，老練的狂奔者都知道這是個好主意。

矩陣全防禦（Full Matrix Defense）：如果你的通訊鏈或任何與之相連的設備受到攻擊，你不知道該怎麼做，就用這個動作開啟通訊鏈的所有防禦手段。

矩陣搜索（Matrix Search）：通信之外，搜索是口袋裡有矩陣的最大優勢。

發送信息（Send Message）：這個動作可能是你通訊鏈最主要的用途。它並不複雜，但你可以去看一眼。

暴力破解 BRUTE FORCE 

（複雜動作）

權限需求：無

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 意志＋防火牆

你可以用這個矩陣動作去給你本不具有權限的一個目標打上MARK。較之〈後台入侵Hack on the fly〉，採用〈暴力破解〉的黑客們更強調其行為的攻擊性，而不是隱匿性。

如果你在對抗檢定中勝出，你在目標上放置１個MARK，對同一個目標最大能擁有３個MARK。如果你希望的話，你在對抗檢定中獲得的每2個淨成功可以讓你對該設備造成1點矩陣傷害。這個傷害值的抵抗檢定為，該設備的DR＋防火牆。

在擲骰前，你可以聲明嘗試同時放置超過一個MARK。如果你試圖在一個動作中放置2個MARK，你會受到-4骰池罰值；如果試圖一次放置3個MARK，那麼骰池罰值變為-10。

你也可以用這個矩陣動作去翻到本不具有合法權限的網域裡面去。在這種情況下，本地網域的防禦骰池是4個骰子，全球網域的防禦骰池是6個骰子。如果你勝出，你不會在網域上放置MARK，而是立刻翻到目標網域中。成功使用〈暴力破解〉以翻越網域不會像大部分成功的攻擊動作一樣觸發網域的警報，也不會驚動次級GOD。

 

改變圖標 CHANGE ICON

（簡單動作）

權限需求：所有者（Owner）

檢定：無（數據處理類動作）

你改變目標的圖標，使其成為一個你擁有的樣本的複製品，或一個你自己設計的圖案。改變圖標並不會改變〈矩陣偵察Matrix Perception〉動作的結果，但可能會蒙蔽那些沒有花時間觀察你的新外表的化身們。你可以把自己的圖標作為〈改變圖標〉行動的目標。

 

查詢監視指數 CHECK OVERWATCH SCORE

（簡單動作）

權限需求：無

檢定：電子戰＋邏輯【隱匿性】v. 6

你可以查明網域離開始對你發動合圍（Converging）還有多遠。查看監視指數（OS）是一個隱匿動作，因此對抗檢定中對方獲得的成功數會加入到你的OS中。如果你在檢定中勝出，GM會告知你開始這個動作時的OS，然後再把防禦骰池中的成功數加到你的OS上。

 

控制設備 CONTROL DEVICE 

（複雜動作）

權限需求：1(自由動作)／2(簡單動作)／3(複雜動作)

檢定：（視情況）【數據處理(/特殊，見詳述)】v. （視情況），或

電子戰＋直覺【隱匿性】v. 直覺＋防火牆

通過像使用遙控器或遊戲機手柄一樣操作你的數據板或通訊鏈，你可以操縱你控制（或有足夠控制權）的一台設備完成一個動作。你嘗試進行的行動所需的檢定骰池使用你在正常情況下進行此項檢定所需的技能等級和屬性等級。舉例來說，

操縱無人機搭載的武器朝目標開火要進行檢定，炮術＋敏捷；

操縱一台遙控式水下焊接機器人需要進行檢定，航海機械＋邏輯。

控制設備期間進行的檢定應用該檢定的正常檢定上限，或你的數據處理等級，取其較低者。如果沒有適合你想讓設備進行的檢定類型，比如解鎖一個磁力鎖，或者彈出手槍的彈匣，你必須進行贏得對抗檢定，電子戰＋直覺【隱匿性】v. 直覺＋防火牆，而不必繼續做出其他檢定。

你可以使用這個矩陣動作同時操控複數台設備。如果每一台設備都為你所擁有，且它們都被命令在同一時間進行相同的行動，那麼這不需要額外花費一些什麼；否則，你必須按照你想在一個動作中操縱的設備個數來分裂骰池。

這個動作的類型（比如，自由動作/簡單動作/標準動作/複雜動作）與受控設備嘗試進行的動作相同，其所需要的權限為：自由動作1個MARK，簡單動作2個MARK，複雜動作3個MARK。

當你在〈控制設備〉的檢定中使用隱匿性來作為檢定上限時，〈控制設備〉成為隱匿動作，這會帶來與一切隱匿動作相同的風險——OS指數的上升，以及失敗所造成的結果。（譯註：隱匿動作失敗會讓你的目標獲得一個對你的MARK，設備會通知其所有者，主機會運行IC反侵略。）

 

文件解密 CRACK FILE

（複雜動作）

權限需求：1個MARK

檢定：駭入＋邏輯【攻擊性】v. 保護等級×2

你去除掉目標文件上附加的保護措施，使其成為可讀取的文件。當然，不需要對一個沒有保護的文件執行這項動作。

 

程序破壞 CRASH PROGRAM

（複雜動作）

權限需求：1個MARK

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 直覺＋防火牆

你將目標的儲存空間部分過載，以此來干擾它正在運行的程序。你需要明確指出哪一個程序是你想要破壞的。你可以通過〈矩陣偵察〉動作，或是通過觀察正在生效的程序效果來確定你的目標究竟在運行哪些程序。如果你在對抗檢定中勝出，該程序被終止，並且不能在該設備上再次運行，直到該設備進行一次重啟。

 

數據棘刺 DATA SPIKE

（複雜動作）

權限需求：無

檢定：矩陣戰鬥＋邏輯【攻擊性】v. 直覺＋防火牆

你向一個化身或是一台設備發射一段有害的指令，由此對目標造成矩陣傷害。該次攻擊的基礎DV為｛攻擊性｝；每個淨成功使DV＋１，以及每個你對目標擁有的MARK使DV＋２。這項傷害是矩陣傷害（見228頁），抵抗檢定為，目標的DR＋防火牆。

 

解除數據炸彈 DISARM DATA BOMB

（複雜動作）

權限需求：無

檢定：軟件＋直覺【防火牆】v. 數據炸彈等級×2

你嘗試解除一個你已經偵測到（通常作為〈矩陣偵察〉檢定的結果）的數據炸彈。如果你得到了至少１個淨成功，那麼數據炸彈被安全地解除並刪除。否則，數據炸彈會被觸發，造成它所設定好的傷害，還有可能會摧毀它所保護的文件（如果這顆炸彈是設計用來摧毀文件的話）。

 

文件編輯 EDIT FILE 

（複雜動作）

權限需求：1個MARK

檢定：計算機＋邏輯【數據處理】v. 直覺＋防火牆

〈文件編輯〉允許你創造、篡改、複製、刪除或保護任意種類的文件。這個檢定的防禦者是持有這個文件的主機或文件的所有者。每個〈文件編輯〉動作可以改變目標文件的一處細節——一條文本的一小段，一幅圖像的一個角落，或者是視頻音頻中的兩三秒。你可以與GM一起協商「一處細節」究竟包括哪種程度。GM可能會因為你做出的編輯過於複雜或棘手而在你的檢定上應用罰值。如果你希望進行連續性的編輯，比如從一段視頻上抹掉隊友們的身影，那麼你需要在每個戰鬥輪中進行一次〈文件編輯〉，直到你停止繼續編輯為止。

如果你用這個動作複製一份文件，那麼你會成為新複製的文件的所有者。如果你嘗試複製的文件具有保護措施，那麼〈文件編輯〉自動失敗。如果這份文件設有數據炸彈，那麼炸彈會在你身上炸開花（所以務必記住在打開文件之前掃瞄它們）。

如果你是文件的所有者，你也可以用〈文件編輯〉來為你的文件設置保護。如果你希望保護一份文件，做一個計算機＋邏輯【數據處理】的簡單檢定，你得到的成功數成為受保護文件的等級。一個受保護的文件不能被讀取、更改、刪除或複製，除非保護被破壞。（譯註：去除文件的保護，需要使用〈文件解密〉矩陣動作。得知其是否被保護需要進行〈矩陣偵察〉動作。）

（複雜動作）

權限需求：1個MARK

檢定：無

你進入你擁有MARK的一台主機，你的圖標會出現在那裡面；你也可以用這個動作從一個主機中離開。這並不需要檢定：主機允許得到MARK的人任意進入，已在內部的人也可以隨意退出。但當你駭入主機時，它可能不會這麼熱情好客。當然，有些IC（反入侵組件）可以把你困在主機裡。那時除非強行闖出，否則你無法使用此動作離開主機。

當你離開主機時，你回到進入主機之前你所在的網域。

你可以跳過正常合法的步驟，用這個動作來強行在一個目標上放置MARK。與〈暴力破解〉相比，使用〈後台入侵〉的黑客們更注重她們的隱匿性，而不是破壞力。

你也可以用這個矩陣動作去翻到本不具有合法權限的網域裡面去。在這種情況下，本地網域的防禦骰池是4個骰子，全球網域的防禦骰池是6個骰子。如果你勝出，你不會在網域上放置MARK，而是立刻翻到目標網域中。依此法使用〈後台入侵〉並在檢定中失敗並不會如同其他失敗的隱匿類動作一樣驚動網域或次級GOD。

這個用途廣泛且重要的動作可以用於兩個方面：尋找矩陣中的圖標，或是分析矩陣中的對象。當你使用〈矩陣偵察〉分析矩陣中的對象或是掃瞄附近的靜默運行中的圖標時，你需要做一個簡單檢定來決定你能獲取多少情報。每個淨成功數可以讓你提出一個關於目標的問題——這可能是目標的類型、等級、MARK數量、其儲存的文件、其接入的網域、區域內是否有靜默運行的圖標，或者其他任何相關的矩陣信息。每個淨成功數會讓你知道一項事實。

設備重啟 REBOOT DEVICE

（複雜動作）

權限需求：3個MARK

檢定：計算機＋邏輯【數據處理】v. 意志＋防火牆

被執行此動作的設備會立刻關機並重新啟動。在下一個戰鬥輪結束時該設備會重新上線。在它重啟完成之前，此設備失去一切電子功能，並從矩陣網絡中消失。

當你重啟自己化身對應的設備時，你的OS重置為0，你的一切MARK（無論是你放置在他處的，還是其他人放置在你圖標上的）全部被抹去。如果你在自己處於VR狀態時重啟，你會承受甩致休克（見229頁）。當你返回線上時，你的圖標可能位於任何一個你擁有合法接入權限的網域中，或者，如果沒有任何網域權限的話，處於公共網域之中。

當你執行這一行動時，你可以選擇讓目標設備關機後延遲任意長短的時間，之後再執行啟動工作。任何一個與該設備有物理接觸的人都可以強行結束這種延遲，只要他們按一下電源鍵就可以讓設備開始運行，並在下一個戰鬥輪結束時讓該設備連接上線。

這個動作只對設備和你自己的化身生效，但對主機、生命體（如超鏈者，儘管他們可以「重啟」自己，見251頁）、共鳴組構體（如網精）無效。

如果你是設備的擁有者，你不需要進行檢定。

你不能對被〔鏈接鎖定〕的設備使用〈設備重啟〉。

 

發送信息 SEND MESSAGE

（簡單動作）

權限需求：不適用（N/A），或1個MARK

檢定：無（數據處理類動作）

你通過矩陣網絡向一個你擁有其通訊碼的用戶發送一段信息。該信息可以是文字或音頻組成的一個短句，也可以是一幅圖像或是一個文件。如果你通過直接神經界面（DNI）接入矩陣——即使你只是在使用AR模式——那麼就可以傳送更長或更複雜的信息，比如一長段文字。你也可以使用這個動作來為一個或多個參與者直播（Live feed），你們可以通過任何一種數字錄製設備來達成這個目的。（譯註：即，通過矩陣共享攝像頭裡的畫面或者數字錄音機傳來的音像等進行直播，或者其他的發送/接收設備傳遞氣味、觸感等。碟客可以利用這個來建個聊天室/討論組/群。）

 

設置數據炸彈 SET DATA BOMB

（複雜動作）

權限需求：1個MARK

檢定：軟件＋邏輯【隱匿性】v.（設備等級×2）

你在一份文件中設置一個數據炸彈。當你進行這個動作時，你需要為數據炸彈設置等級，最高等級為你在檢定中獲得的淨成功。你還需要選擇炸彈引爆時是否會刪除掉它所附屬的文件。最後，為你的數據炸彈設置一個密碼，輸入密碼可以避免炸彈的觸發。一份文件同時只能設置一個數據炸彈。

當某人試圖在未輸入已設置好的密碼的情況下閱讀、編輯、拷貝、保護、刪除該文件，或在該文件中設置另一個數據炸彈時，炸彈就被觸發。

數據炸彈的引爆會對落入陷阱的圖標造成｛數據炸彈的等級個D6｝點矩陣傷害（如常抵抗）。按照設置的不同，爆炸的炸彈還可能會刪除該文件，然後刪除它自己。如果在進行上述會觸發炸彈的行動之前輸入了正確的密碼，那麼數據炸彈不會啟動，但依然附著在該文件之上，等待著下一個使用者。（譯註：大寫D6的時候，得出來的值要全部加在一起計算總值，而不是計算成功值。矩陣火球術）

數據炸彈可以通過〈矩陣偵察〉動作被偵測到。如果它被偵測到了，其他人就可以用〈解除數據炸彈〉動作來使其無力化。一個被解除的數據炸彈會被刪除掉，該死的。

 

窺探 SNOOP

（複雜動作）

權限需求：1個MARK

檢定：電子戰＋直覺【隱匿性】v. 邏輯＋防火牆

這個動作使你可以攔截目標設備發送到矩陣和從矩陣接收到的數據流，只要你依然對其擁有MARK。你可以現場收聽、觀看或閱讀竊取到的數據，也可以保存起來以便之後回放/查閱，只要你有用來儲存它們的設備——你的碟板樂意效勞。

 

窺探的用途USES OF SNOOP

〈窺探〉的用途常常不僅僅止步於偷看偷聽。如果你的目標在打電話（make a commcall），你可以記錄到他聯繫的目標，並通過一個〈矩陣偵察〉動作來在矩陣網絡上找到他們。如果他們在100米以內，你立刻在矩陣中發現他們。（如果你所窺探目標的聯繫人正在靜默運行的話，你必須按〈矩陣偵察〉的規則對抗並取得淨成功）

 

偽造指令 SPOOF COMMAND

（複雜動作）

權限需求：1個MARK（見描述）

檢定：駭入＋直覺【隱匿性】v. 邏輯＋防火牆

你欺騙一台設備的所有者認證系統，以使這台設備認為你的指令是來自於其所有者的合法指令。你需要對你模仿的圖標（即該設備的真正所有者——譯注）擁有1個MARK，但並不需要對目標設備擁有MARK。儘管如此，對抗骰池依然是基於該設備的屬性。這個動作只對設備或代理（Agents）生效，對IC、網精、主機、化身或其他任何圖標都無效。

 

切換界面模式 SWITCH INTERFACE MODE 

（簡單動作）

權限需求：所有者

檢定：無（數據處理類動作）

你把你的視角從AR模式切換到VR模式，或者反過來從VR模式切換到AR模式。切換到VR模式會使你的身體不能動彈，所以別在危險的地方這麼幹。如果你從VR模式切換到AR模式，你失去VR模式下的獎勵主動性骰，見【改變主動性】。如果你被鏈鎖（229頁），你不能切換切入模式。

你只能切換自己的接入模式，不能對其他人這麼幹。

 

圖標追蹤 TRACE ICON

（複雜動作）

權限需求：2個MARK

檢定：計算機＋直覺【數據處理】v. 意志＋隱匿性

你尋找矩陣中目標設備或化身的物理定位。在對抗檢定中勝出後，只要你依然對目標擁有至少一個MARK，你就可以找到它的位置。〈圖標追蹤〉對主機無效，因為它們一般沒有物理位置；〈圖標追蹤〉同樣無法作用於IC，因為它們只存在於它們的主機之內。（用來順著網線砍人）
